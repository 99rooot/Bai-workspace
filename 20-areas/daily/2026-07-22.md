Slack 봇 코드만 Git 커밋하고 GitHub에 올렸다.

Vercel에서는 다음 폴더만 배포 대상으로 선택했다.

실제 값은 코드, GitHub, 보고서에 기록하지 않았다.


일반 주소를 열었을 때 `404`가 나오는 것은 정상이다. 이 프로젝트는 홈페이지가 아니라 `/slack/events`라는 Slack 전용 API만 제공한다.

### 연결과 설정이 필요했던 이유

#### Codex와 GitHub 연결을 시도한 이유

Codex가 만든 코드를 GitHub에 직접 올리고, 그 변경을 Vercel이 감지해 Slack 봇을 자동으로 다시 배포하게 만들기 위해서였다.

```text
Codex가 코드 수정
→ GitHub에 코드 업로드
→ Vercel이 변경 감지
→ Slack 봇 자동 배포
```

하지만 Codex GitHub 플러그인의 권한은 이 맥만이 아니라 Codex에 로그인한 계정에 연결될 수 있었다. 같은 계정을 다른 사람이 사용할 때 GitHub 저장소 접근 범위가 넓어질 가능성을 걱정해 이 연결은 해제했다.

대신 이 맥에만 저장한 SSH 개인키로 GitHub에 코드를 올리는 방법을 선택했다. 이 방법은 Codex 계정에 GitHub 권한을 남기지 않으면서도 현재 맥에서는 계속 배포할 수 있다는 장점이 있다.

#### Vercel과 GitHub 연결을 유지한 이유

Vercel은 GitHub의 `main` 브랜치가 변경되면 새 코드를 자동으로 배포해야 한다. 따라서 Codex GitHub 플러그인은 해제했지만, Vercel과 `Bai-workspace` 저장소의 연결은 자동 배포를 위해 유지했다.

#### Slack과 Vercel 연결이 필요한 이유

Slack은 휴대폰에서 명령을 보내고 답장을 받는 창구이고, Vercel은 맥이 꺼져 있어도 Slack 메시지를 받아 버스 정보를 계산하는 실행 공간이다. Slack에는 Vercel의 `/slack/events` 주소를 등록했고, Vercel에는 Slack 요청을 확인하고 답장을 보내는 데 필요한 비밀값을 환경변수로 저장했다.

### 막힌 점과 해결

#### Vercel에 하위 폴더가 보이지 않음

GitHub 저장소가 실제로 비어 있어 Vercel이 루트 폴더만 표시했다. GitHub 플러그인의 완료 응답만 믿지 않고 공개 GitHub API로 다시 확인하면서 문제를 찾았다.

해결 방법:

1. 이 맥 전용 SSH 키 생성
2. GitHub에 공개키 등록
3. 로컬 Git 원격 주소를 SSH 방식으로 변경
4. `git push` 실행
5. GitHub 공개 API에서 `10-projects` 폴더 확인

#### 새벽에는 버스 정보가 없음

새벽에는 실제 운행 정보가 없어서 봇이 제한적인 답을 보냈다. 이는 배포 오류가 아니라 운행 시간이 아니기 때문에 생기는 정상적인 결과다.

### 오늘 배운 것

#### 로컬 서버와 클라우드 서버의 차이

- 로컬 서버: 내 맥이 켜져 있고 같은 네트워크에 있어야 한다.
- 클라우드 서버: Vercel처럼 인터넷에 있는 컴퓨터가 대신 실행하므로 장소와 Wi-Fi에 영향을 덜 받는다.

#### 주소 연결과 로그인 권한은 다르다

Git에 GitHub 저장소 주소를 등록했다고 바로 업로드할 수 있는 것은 아니다. 주소는 목적지만 알려주고, SSH 키나 로그인 토큰이 실제 신분증 역할을 한다.

#### 공개키와 개인키

- 공개키: GitHub에 등록해도 되는 자물쇠 정보
- 개인키: 내 맥에만 보관해야 하는 열쇠

개인키를 가진 이 맥만 GitHub에 인증할 수 있다.

#### 환경변수

Slack 토큰 같은 비밀값을 코드에 적지 않고 Vercel 설정에 따로 저장하는 방법이다. 코드가 공개되어도 비밀값은 공개되지 않는다.
